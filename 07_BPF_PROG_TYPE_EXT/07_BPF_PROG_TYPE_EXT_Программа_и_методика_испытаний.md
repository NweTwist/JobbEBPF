__BPF\_PROG\_TYPE\_EXT__

ПРОГРАММА И МЕТОДИКА ИСПЫТАНИЙ

Исполнитель: \_\_\_\_\_\_\_\_\_\_\_\_\_\_\_

__1 Объект испытаний__

Объект испытаний представляет собой программу «BPF\_PROG\_TYPE\_EXT», реализующую тип eBPF\-программ BPF\_PROG\_TYPE\_EXT\.

Обозначение программы 07\-EXT\-BPF \(файл target\.bpf\.c\)\.

__2 Цель испытаний__

Целью испытаний является проверка работоспособности программы типа BPF\_PROG\_TYPE\_EXT на отечественных операционных системах Astra Linux 1\.8\.3 и RED OS 8\.0\.2\.

__3 Требования к программе__

В ходе испытаний подлежат проверке следующие требования\.

1\) компиляция программы из исходного кода в BPF\-объект на целевых платформах;

2\) загрузка BPF\-объекта в ядро ОС и прохождение верификации;

3\) корректное выполнение функций программы;

4\) корректное отключение программы и удаление из ядра;

__4 Требования к программной документации__

На испытания предъявляется следующая документация\.

1\) техническое задание;

2\) описание программы;

3\) текст программы;

4\) настоящая программа и методика испытаний\.

__5 Средства и порядок испытаний__

__5\.1 Технические средства__

__5\.2 Программные средства__

1\) компилятор clang/LLVM;

2\) утилита bpftool;

3\) заголовочный файл vmlinux\.h;

4\) библиотека libbpf;

5\) компилятор gcc \(для загрузчика пространства пользователя\)\.

Установка необходимых пакетов на Astra Linux 1\.8\.3\.

apt install \-y clang llvm linux\-tools\-6\.1 libbpf\-dev libelf\-dev zlib1g\-dev gcc

Установка необходимых пакетов на RED OS 8\.0\.2\.

dnf install clang llvm bpftool libbpf\-devel elfutils\-libelf\-devel zlib\-devel gcc

__5\.3 Порядок проведения__

Испытания проводятся последовательно на каждой виртуальной машине QEMU\. Испытания выполняются от имени пользователя root\.

__6 Методы испытаний__

Испытания проводятся последовательно выполнением приведённых ниже команд от имени пользователя root\.

1\) Генерация заголовочного файла vmlinux\.h из BTF\-информации ядра\.

bpftool btf dump file /sys/kernel/btf/vmlinux format c > vmlinux\.h

2\) Компиляция целевой ядерной BPF\-программы target\.bpf\.c в объектный файл\.

clang \-O2 \-g \-target bpf \-c target\.bpf\.c \-o target\.bpf\.o

3\) Генерация скелета libbpf из объектного файла target\.bpf\.o\.

bpftool gen skeleton target\.bpf\.o > target\.skel\.h

4\) Компиляция BPF\-программы расширения ext\_replace\.bpf\.c в объектный файл\.

clang \-O2 \-g \-target bpf \-c ext\_replace\.bpf\.c \-o ext\_replace\.bpf\.o

5\) Генерация скелета libbpf из объектного файла ext\_replace\.bpf\.o\.

bpftool gen skeleton ext\_replace\.bpf\.o > ext\_replace\.skel\.h

6\) Компиляция программы пространства пользователя\.

gcc \-O2 \-o ext\_test ext\_test\.c \-lbpf \-lelf \-lz

7\) Запуск программы\.

\./ext\_test

8\) Проверка вывода программы\. Ожидаемый результат\.

  \[LOAD\] OK

  \[ATTACH\] OK

  \[TRIGGER\] OK

  \[VERIFY\] PASS \(counter=N\)

  \[CLEANUP\] OK

Критерий прохождения: вывод программы содержит строку «\[VERIFY\] PASS»\.
